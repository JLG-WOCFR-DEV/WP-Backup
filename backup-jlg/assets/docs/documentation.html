Guide de démarrage Backup JLG

Tout ce qu’il faut pour configurer vos premières sauvegardes et sécuriser votre site WordPress.

Étape 1 — Préparer l’environnement

  • Activez le plugin puis ouvrez Backup JLG → Réglages pour vérifier les prérequis (version de PHP, taille mémoire, Cron).
  • Définissez la clé de chiffrement : Backup JLG → Chiffrement → Générer une clé puis lancez le test d’intégrité.
  • Créez au besoin une clé API pour vos automatisations CI/CD dans API & Intégrations.

Étape 2 — Lancer la première sauvegarde

  • Depuis l’onglet Sauvegarde & restauration, choisissez les composants à inclure (fichiers, base de données, uploads…).
  • Activez l’option Sauvegarde chiffrée pour générer une archive sécurisée AES-256.
  • Démarrez la sauvegarde : la progression s’affiche en temps réel et chaque action est journalisée.

Étape 3 — Automatiser et surveiller

  • Planifiez des sauvegardes récurrentes dans Réglages → Planification en choisissant fréquence, destinations et rotation.
  • Connectez une destination distante (S3, Google Drive, SFTP…) pour externaliser les archives.
  • Consultez le tableau de bord : indicateurs clés, alertes et recommandations apparaissent instantanément.

Ressources complémentaires

Le plugin embarque une suite d’outils pour affiner votre stratégie :

  • Nettoyage automatique : purge quotidienne des archives obsolètes et des fichiers temporaires.
  • API REST complète : gérez les sauvegardes, restaurations et métriques depuis vos scripts.
  • Bloc Gutenberg : exposez l’état des sauvegardes directement dans votre intranet.
