Planifier ses sauvegardes en 5 minutes

Suivez ce tutoriel pas à pas pour configurer une stratégie de sauvegarde automatique robuste avec Backup JLG.

1. Définir la fréquence idéale

  1. Ouvrez Backup JLG → Réglages → Planification.
  2. Créez un nouveau scénario ou dupliquez un modèle existant pour partir d’une configuration fiable.
  3. Choisissez la fréquence adaptée : horaire pour les sites très actifs, quotidienne ou hebdomadaire sinon.
Astuce : combinez une sauvegarde quotidienne incrémentale avec une sauvegarde complète hebdomadaire.

2. Sélectionner les composants à sauvegarder

  1. Activez les éléments indispensables : base de données, uploads, thèmes et plugins.
  2. Ajoutez des filtres d’exclusion (cache, exports volumineux…) pour accélérer le traitement.
  3. Définissez un nom explicite afin d’identifier rapidement le scénario dans l’historique.

3. Activer le chiffrement et la rotation

  1. Dans l’onglet Options avancées, cochez Activer le chiffrement AES-256.
  2. Renseignez un mot de passe de récupération (4 caractères minimum) si l’archive doit être protégée.
  3. Configurez la rotation : par exemple conserver les 10 dernières sauvegardes ou les archives des 30 derniers jours.

4. Envoyer les archives vers une destination distante

  1. Activez la destination souhaitée (S3, Google Drive, SFTP, Backblaze B2…).
  2. Renseignez les identifiants requis puis utilisez le bouton Tester la connexion.
  3. Définissez la stratégie : envoi systématique ou uniquement pour les sauvegardes complètes.
Besoin d’un contrôle fin ? Combinez plusieurs destinations et activez les notifications Slack/Webhook.

5. Vérifier le suivi et les alertes

  1. Validez le scénario : la première exécution est planifiée automatiquement.
  2. Retrouvez la prochaine exécution et le taux de succès dans le tableau de bord de Backup JLG.
  3. Consultez l’onglet Diagnostics & logs pour inspecter les journaux détaillés en cas d’incident.
Pour surveiller votre infra à distance, exposez le bloc Gutenberg « État des sauvegardes » sur votre intranet.

Aller plus loin avec WP-CLI

Intégrez la planification à vos pipelines CI/CD :

wp bjlg schedule create --label="Sauvegarde nocturne" \
    --recurrence=daily --components=db,uploads --encrypt \
    --destination=s3 --rotation=10

Votre première sauvegarde planifiée est prête ! Pensez à tester régulièrement la restauration dans un environnement de préproduction.
